A program is a passive set of instructions on disk; a process is an active execution with its own memory and resources. Threads run within a process and share its memory, unlike processes which donâ€™t share memory and must communicate externally.

We can think of monks translating a text: each monk (thread) gets part of the text (shared variable) from an aggregator and returns the translation. If each monk has a vocabulary copy, no further communication is needed.

To manage shared data safely, we use mutexes:

    Init: pthread_mutex_init(&mutex, NULL)

    Lock/Unlock: pthread_mutex_lock/unlock(&mutex)

    Destroy: pthread_mutex_destroy(&mutex)

A user object uses 8 bytes for the pointer and 4 bytes for the int.

Declaring vs defining functions:

    Declaration tells the compiler a function exists.

    Definition provides its actual code.

Using extern before a variable tells the compiler it's defined elsewhere; without it, it looks for a definition.

Libraries are included with #include.
The linker connects function calls with their definitions across different files during compilation.
